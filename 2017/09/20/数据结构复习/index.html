数据结构复习

发表于 2017-09-20 | 阅读次数:

线性表

  • 顺序存储结构
  • 链式存储结构
    • 单链表
    • 静态链表
    • 循环链表
    • 双向链表

栈

  • push压栈、pop弹栈
  • 顺序存储结构
    • 定义栈空间
  • 共享栈空间
    • 使用一个数组存放两个栈，1栈底在数组始端a[0]，2栈底在数组末端a[n-1]，top1 + 1 == top2时栈满
  • O(1)
  • 链式存储结构
    • 不需定义头结点
  • 不需定义栈空间
  • O(1)
  • 递归
  • 中缀表达式转后缀表达式
    1. 将符号压栈，括号匹配时弹栈，弹出括号内符号栈顶元素
    2. 优先级高时，栈中元素全部弹出，新符号压栈
    3. 最后栈中元素全部弹栈
  • 后缀表达式计算
    1. 数字压栈
    2. 遇到符号，栈顶两个数弹栈并计算
    3. 计算结果压栈

队列

  • 顺序存储
    • 入列O(1)，出列O(n)
    • 循环队列
      • 避免入列出列时移动数据
      • O(1)
  • 需考虑队列长度
  • 链式存储
    • O(1)

树

  • 结点
    • 度：结点拥有子树个数
    • 叶结点、分支结点
    • 深度：结点最大层次
  • 存储结构
    • 双亲表示法
    • 孩子表示法
      • 单链表存储孩子结点，顺序存储头指针组成的单链表
        • 孩子链表结点 child next
        • 表头结点 data firstchild
    • 孩子兄弟表示法
  • 二叉树
    • 斜树
    • 满二叉树
    • 完全二叉树
      • 中间无空挡
      • 存储结构
        • 顺序存储只用于完全二叉树
        • 二叉链表 lchild data rchild
      • 遍历
        • 前序遍历
          1. 根节点
          2. 前序遍历左子树
          3. 前序遍历右子树
        • 中序遍历
          1. 中序遍历左子树
          2. 根节点
          3. 中序遍历右子树
        • 后序遍历
          1. 从左到右先叶子后结点遍历左右子树
          2. 根节点
        • 层序遍历
          1. 根节点
          2. 逐层遍历
    • 线索二叉树
      • lchild ltag data rtag rchild
      • tag == 0 孩子标记
      • tag == 1 前驱/后继
  • 哈夫曼树
    • 带权路径长度WPL最小
    • 路径长度：分支数目
    • 树路径长度：根结点到每一结点路径长度之和
    • WPL = Σ(权 * 结点到树根的路径长度)
    • 构造哈夫曼树
      1. 叶结点按权值从小到大排列
      2. 最小权值结点做N1左孩子，第二小做N1右孩子
      3. N1替换以上两结点，插入序列，保证从小到大排列
      4. 重复步骤2
    • 哈夫曼编码
      1. 根据概率构建哈夫曼树
      2. 权值左分支改为0，右分支改为1

  • 网：带权图

  • 无向图

    • 边用()表示
    • 顶点的度（Degree）TD(v)：与顶点相关联边的数目
    • 路径：顶点序列

  • 有向图

    • 边用<>表示
    • 入度 ID(v)、出度OD(v)。TD(v) = ID(v) + OD(v)

  • 存储方式

    • 邻接矩阵：一维数组存顶点，二维数组存边（或弧）
    • 邻接表：一维数组存顶点，线性表存顶点的邻接点

  • 遍历

    • 深度优先遍历DFS
      • 相当于树的前序遍历
    • 广度优先遍历BFS
      • 相当于树的层序遍历

  • 最小生成树

    • Prim

第 4 步 × × × 1 {3, 1, 2, 4}
第 5 步 × × × × {3, 1, 2, 4}
  • Kruskal

  • 最短路径

    • Dijkstra
    • Floyd

查找

  • 顺序表查找
    • 顺序查找
      • 哨兵
  • 有序表查找
    • 二分查找
      • 不便于动态查找
    • 插值查找
      • 将1/2改为(key - a[low]) / (a[high] - a[low])
      • 不适合极端不均匀数据
    • 斐波那契查找
  • 线性索引查找
    • 稠密索引：索引数据集一一对应
    • 分块有序
      • 块内无序
      • 块间有序
    • 倒排索引（搜索技术常用）
  • 二叉排序树
    • 动态查找后，对二叉树进行中序遍历，得到有序序列
  • 平衡二叉树
    • 平衡因子（Balance Factor）：左子树深度减右子树深度
    • BF正右旋，负左旋
    • 符号不统一时，先按1值旋，统一符号
  • 多路查找树：结点孩子数可大于2，结点可储存多个元素
    • 2-3树
    • 2-3-4树
    • B树
      • 阶（Order）：结点对大的孩子数目
