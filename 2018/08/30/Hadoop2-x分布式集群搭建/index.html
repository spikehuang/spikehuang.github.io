Hadoop2.x分布式集群搭建

发表于 2018-08-30 | 阅读次数:

准备工作

  1. 在虚拟机上搭建四个节点，各节点信息如下表所示

  2. 修改各节点/etc/hosts

  3. 保证各节点间可以ping通

  4. 各节点解压jdk-8u151-linux-x64.tat.gz至/usr/local/java/，并配置环境变量

  5. 各节点解压hadoop-2.6.5.tar.gz至/usr/local/hadoop/

  6. 配置node1 ssh免密登录至node2、node3、node4（参考https://blog.csdn.net/mmd0308/article/details/73825953）

hostname ipaddress role
node1 192.168.1.141 Active NameNode
node2 192.168.1.142 Standby NameNode
JournalNode
DataNode
node3 192.168.1.143 Actice NameNode
JournalNode
DataNode
node4 192.168.1.144 Standby NameNode
JournalNode
DataNode

修改Hadoop配置文件

在node1修改以下配置文件，再同步到其他各节点即可

cd /usr/local/hadoop/hadoop-2.6.5/etc/hadoop/

hadoop-env.sh

vim hadoop-env.sh

core-site.xml

vim core-site.xml

mapred-site.xml

vim mapred-site.xml

yarn-site.xml

vim yarn-site.xml

fairscheduler.xml

由于在yarn-site.xml中配置调度策略为FairScheduler，需要新建fairscheduler.xml自定义配置文件

vim fairscheduler.xml

hdfs-site.xml

vim hdfs-site.xml

slaves

vim slaves

同步至各节点

scp ./* root@node2:/usr/local/hadoop/hadoop-2.6.5/etc/hadoop/

scp ./* root@node3:/usr/local/hadoop/hadoop-2.6.5/etc/hadoop/

scp ./* root@node4:/usr/local/hadoop/hadoop-2.6.5/etc/hadoop/

这里需要注意的是，node3和node4作为hadoop-cluster2，需要更改hdfs-site.xml配置文件

启动Hadoop集群

cd /usr/local/hadoop/hadoop-2.6.5/

启动node1、node2

启动JournalNode

在各个JournalNode节点（node2、node3、node4）上，输入以下命令启动JournalNode服务

sbin/hadoop-daemon.sh start journalnode

在node2、node3、node4上输入jps指令，若存在JournalNode进程，说明JournalNode启动成功

启动NameNode

在node1上格式化NameNode

bin/hdfs namenode -format -clusterId hadoop-cluster

启动NameNode

sbin/hadoop-daemon.sh start namenode

同步元数据信息

在node2上，同步node1的元数据信息

bin/hdfs namenode -bootstrapStandby

启动NameNode

sbin/hadoop-daemon.sh start namenode

切换node1为active

经过以上步骤，node1、node2均处于standby，这里我们还没有配置ZooKeeper，Hadoop集群不能自动选举出主节点，需要手动将节点切换至active状态

bin/hdfs haadmin -ns hadoop-cluster1 -transitionToActive nn1

这时，访问http://192.168.1.141:50070、http://192.168.1.142:50070返回以下页面，说明node1、node2启动成功

启动node3、node4

启动NameNode

在node3上格式化NameNode

bin/hdfs namenode -format -clusterId hadoop-cluster

这里注意clusterId要与node1启动命令中的clusterId一致（这里node1、node3均为hadoop-cluster）

启动NameNode

sbin/hadoop-daemon.sh start namenode

同步元数据信息

在node4上，同步node3的元数据信息

bin/hdfs namenode -bootstrapStandby

启动NameNode

sbin/hadoop-daemon.sh start namenode

切换node3为active

bin/hdfs haadmin -ns hadoop-cluster2 -transitionToActive nn3

启动所有DataNode

在node1上，启动所有节点DataNode

sbin/hadoop-daemons.sh start datanode

这时访问每个节点的50070端口，都可以看到Live Nodes值为3，说明DataNode已经启动成功

启动Yarn

在node1节点启动Yarn

sbin/start-yarn.sh

访问http://192.168.1.141:8088，会返回以下页面，至此整个Hadoop集群启动成功

配置环境变量

为了更方便地使用hadoop、hdfs指令，可以配置Hadoop环境变量

vim ~/.bashrc

输入以下命令使其生效

source ~/.bashrc

这时直接在命令行输入hadoop version测试Hadoop环境变量是否生效

测试程序

我们使用经典的WordCount程序测试Hadoop集群

可以直接使用Hadoop提供的示例jar包（位于hadoop-2.6.5/share/hadoop/mapreduce/hadoop-mapreduce-examples-2.2.0.jar）进行测试，也可以自己编写MapReduce程序打包成jar文件再放到Hadoop集群上运行，这里选择第二种方法

我使用了IDEA、Gradle搭建learn-hadoop项目，gradle及WordCount类代码如下

将项目打jar包，传输到node1节点上

WordCount程序还需要准备输入文件，这里随便选择一篇文章，上传到HDFS中/user/spike/input/目录下

hdfs dfs -mkdir -p /user/spike/input

hdfs dfs -put article.txt /user/spike/input/

执行程序

hadoop jar learn-hadoop-1.0-SNAPSHOT.jar io.spike.mapreduce.WordCount /user/spike/input/ /user/spike/output/

执行成功后，可以查看到HDFS上的输出文件

踩坑

在格式化NameNode时报错java.net.NoRouteToHostException: 没有到主机的路由，经查看是没有关闭防火墙，可参考https://blog.csdn.net/lsxy117/article/details/47217161。我搭建的各个节点运行在CentOS7上，使用`systemctl stop firewalld.service`关闭防火墙。

总结

搭建Hadoop分布式集群是学习后续大数据知识的必需工作，由于之前已经搭建过单点Hadoop系统，在这次搭建过程中并没有遇到太大的麻烦。
